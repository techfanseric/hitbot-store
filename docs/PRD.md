## 1. 项目概述

### 1.1 一句话定义
**HitbotOS 自营 B2B 商城**——挂在 HitbotOS 官网上、面向企业客户的零件/组件直营购买渠道，与 OS 软件端共用一套购物车/订单数据。

### 1.2 项目边界（in scope / out of scope）

| In scope（本期做） | Out of scope（本期不做） |
|---|---|
| 官网商品展示/详情/分类/购物车/结算 | 第三方卖家入驻（marketplace） |
| 个人中心 + B2B 三级账号 | 公开用户注册（必须企业邀约） |
| OS 端（设备库 + 3D 仿真）购物车入口 + 一键下单 | OS 端的"纯体验入口"中的购买流程 |
| 管理后台：商品、订单、权限 | 物流配送系统对接、退款流程细节 |
| 官网"在线体验 OS"入口（手机号体验账号） | 体验账号转正流程 |
| 视觉规范：颜色表、去边框 | 完整设计系统（只出表格） |
| 嵌入到即将上线的新官网 |  |

### 1.3 关键判断
- **商品分类**遵循 OS 系统里的产品分类（即大族/Hitbot 自家的机器人、夹爪、模组等硬件）
- **不沿用任何电商开源模板**——评估过现有模板都是半成品，按自家需求从零搭，工作量评估 1-2 周
- **官方流程就放在官网**，不要放在 OS 客户端管理后台

## 2. 用户与账号体系

### 2.1 角色定义

| 角色 | 入口 | 关键权限 |
|---|---|---|
| **企业账号（管理员）** | 官网/管理后台 | 创建/管理下级账号；管理团队设备库、模型库；查看/审批订单 |
| **采购** | 官网商城 | 结算、付款（对公/对私）、开发票、订单审批；**不在 OS 端角色中** |
| **工程师** | OS 软件端 + 官网 | 设备库设计、**只加购不结算**；一键下单（跳转官网给采购接续） |

### 2.2 账号模型参考
参考"嘉立创交易商"的三级体系：
```
企业账号 (公司主体)
   ├─ 采购账号（结算、付款、开发票）
   └─ 工程师账号（设计、加购、提交订单申请）
```

### 2.3 OS 端账号体系
- OS 端默认有"工程师"账号（**默认**身份）
- 同一企业下，工程师有"管理员"角色：可管理团队设备库/模型库，添加/修改
- 采购不在 OS 端角色中

### 2.4 登录态策略（官网）
- **浏览商品**：不强制登录
- **加购物车**：要求登录（必须能跨设备同步购物车、记住加购方案）
- **结算**：必须登录
- 参考 Apple Store 模式：浏览 → 加购（登录）→ 结算

## 3. 前台功能（官网 H5 商城）

### 3.1 页面结构
```
官网首页
├─ 产品中心（明码标价，明列全部商品）
│   └─ 选购 → 商品详情（具体型号 + 配置）
│       ├─ 详情图片（多张）
│       ├─ 规格参数
│       ├─ 价格
│       └─ "加入购物车" 按钮
├─ 产品分类（按 OS 系统分类）
│   ├─ 商品列表
│   └─ 筛选 / 排序
├─ 在线体验 OS 入口（详见 §6）
├─ 购物车（全局入口，右上角图标，三角+购物车形态）
│   ├─ 物料清单（区分 标准件 / 加工件 / 定制件 / 参考件）
│   ├─ 勾选要买的（未选 = 不下单，参考件默认排除）
│   ├─ 数量编辑
│   └─ "去结算"
├─ 结算
│   ├─ 确认清单（勾选要下单的物料）
│   ├─ 收货地址
│   ├─ 付款方式（**对公 / 对私**，B2B 核心场景）
│   ├─ 发票信息（对公付款必填：法人、税务号、银行账户等，待定）
│   └─ 收银台
└─ 个人中心
    ├─ 订单列表（按项目）
    ├─ 权限管理（企业管理员可见）
    └─ 账号信息
```

### 3.2 全局购物车规范
- **位置**：右上角，常驻图标（拟采用"三角 + 购物车"形态）
- **数据源**：与 OS 端共用同一套数据库、同一个购物车
- **跨端一致**：在 OS 端加购、在官网查看，反之亦然
- **OS 端入口**：top 图/2D 设备库 页面**和** 3D 仿真页面 都有购物车图标（不只是 3D 仿真）
- **项目语境**：购物车是**当前打开项目**下的 BOM；切项目 → 切换对应购物车内容
- **离线状态**：OS 端可暂存本地（弱网/无网），恢复后同步

### 3.3 物料清单的三分类
- **标准件**（设备库里的成品）—— 直接下单
- **加工件 / 定制件**（型材类，需加工）—— 单独走定制流程
- **环境/参考件**（3D 场景中的桌子、背景等）—— 不卖，仅作展示，用户可勾选排除

UI 上用**颜色**区分类别（不靠边框、底色区分）。

### 3.4 物料清单生成规则（OS 端）
- 用户在 OS 端设计/搭建方案（top 图 + 3D 仿真）后
- "生成购物车/物料清单"按钮：一键将所有涉及的设备转成清单
- 完整列出（包括"我们不卖的"和"客户不要的"）—— 用户自己勾选
- 勾选 = 实际下单项；未勾选 = 仅作展示

### 3.5 个人中心 - 订单
- 按**项目**分组：一个项目对应一个或多个订单
- 订单详情可下载完整物料清单
- 实时显示订单进度（已下单/生产中/已发货/已完成）

## 4. OS 软件端集成

### 4.1 设备库购物车入口
- OS 端"设备库"页面（top 图/2D 布局）右上角加购物车图标
- 工程师在设备库中添加设备 → 同步加入购物车
- 设备库页面的"购物车"图标点击后 → 跳转官网购物车页（保持同一份数据）

### 4.2 3D 仿真 + 设备库 + 购物车的关系
```
项目
├─ 设备库（top 图/2D）
│   └─ 购物车入口（跳转官网）
├─ 3D 仿真（动作编辑）
│   └─ 与设备库共用同一购物车
└─ 全局购物车（右上角）
```

### 4.3 一键下单
- 工程师在 OS 端完成方案设计
- 右上角购物车图标 → "一键下单"
- 跳转官网购物车（保持登录态和数据）
- 采购在官网确认/结算

**下单消息机制**：OS 端"一键下单"本质是**给后端发一个订单消息**（不直接生成订单），由采购在官网接续完成结算与下单动作。OS 端不直接生成订单，避免工程师误操作直接扣企业预算。

## 6. 官网"在线体验 OS"入口（独立需求）

### 6.1 定位
**不涉及账户的纯体验入口**——参考绿联 NAS 体验模式。**核心定位是"引流"**：让客户在购买前能直观感受到 OS 长什么样，解决"现在哪一个客户搞试用他也感受不到这个东西"的问题。

> 用户场景：客户在购买前，想先"看到"我们的 OS 长什么样。

### 6.2 流程
1. 官网首页"OS 介绍"页面
2. 填写手机号 → 获得体验账号（**假项目**，数据退出后清空）
3. 进入"假项目"界面（简化版 OS）
4. 引导式体验，可看不可用

### 6.3 体验版 OS 规范
- **和正式 OS 几乎一致**（保留 UI 一致性，是最重要的边界约束）——**只读 + 引导式体验**的简化版，不是阉割版
- 设备库**只读**（不能拖入大量东西）
- 3D 仿真**可拖拽少量模型**（预设"最小系统"，如基础夹板、夹爪、几个标准物料）
- 右上角功能**disabled 状态**（看到有但点不动）
- 隐藏：管理后台入口、订单、真实购物车
- **不进入新界面**（绿框碰撞体编辑时就在原 3D 场景内直接编辑）
- **不调任何后端**：不下载、不上传数据；不持久化项目状态
- **引导式体验**：用引导步骤带用户走一遍"看 UI、看 3D、看设备库"流程，而非开放自由操作

### 6.4 技术选型
- 简化版 3D：Three.js 实现简版
- Unity Web 版（需要适配，目前只支持 Windows/Linux）
- 本期目标：**加个入口** + 跑通最小体验闭环

### 6.5 优先级
- **中等偏低**（"优先级没有这么高"）
- 但希望**本版迭代里加上入口 + 跑通最小化体验闭环**
- 完整体验闭环可分阶段后续迭代

## 7. 视觉设计规范

### 7.1 颜色
- **统一所有交互组件颜色**（按钮、面板、状态色等）
- 设计师出一份"颜色规范表格"——前端按表格调整
- 用颜色对比替代边框（避免 Unity 引擎渲染细线问题）

### 7.2 边框
- **少用边框**——尽量用**强对比色**区分
- 边框在 Unity 引擎中渲染支持差，必须**基本不用**
- 阴影保留可用

### 7.3 字体/字号/间距
- 待设计师出规范

## 8. 与 OS 现有功能的集成

### 8.1 数据源选择（运行模式）
OS 端仿真运行时，需要选择数据源：
- **3D 场景**（动作编辑器）
- **Blockly**（程序块，可包含多个区域：主区域 + 其他子区域）
- **流程图**

UI 形态：右上角工具栏下拉框（类似腾讯会议麦克风下拉）
- **不要默认选中**（如果 3D 没建程序，默认会让用户运行失败）
- 单选下拉，选中后切换数据源
- 选中 **Blockly** → 展开二级下拉（选区域：主区域 + 其他子区域）
- 其他模式选中后直接关闭二级，无需二级选择

### 8.2 仿真/真机/孪生模式
- 工具栏切换（仿真 vs 真机 vs 孪生，可单选也可"两边一起"）
- 切换后**上面板切换**为「运行/重置」按钮，**其他按钮隐藏**
- 底部"播放器式"方案已讨论被否——不倾向视频播放器交互（会反复显隐，看起来太奇怪）
- 三种模式均支持"单独仿真"、"单独真机"、"两边一起"三种组合

### 8.3 跨端状态同步
- OS 端运行状态变更 → 通过 **MQTT** 同步到所有端
- 两边运行逻辑一致

## 10. 待定/未决问题

1. **退货/退款流程**：会议提及但未细化，本期暂不实现
2. **发票管理**：对公付款涉及，但字段、格式、流程未定
3. **支付通道**：收银台接入哪家（微信/支付宝/银企直连？）
4. **企业账号开通流程**：是否要销售/客服人工开通？自助还是后台？
5. **跨账号数据隔离**：工程师 A 看到的购物车和工程师 B 是否共享？是否按"项目"做隔离（同一企业的工程师共享项目购物车 vs 各自独立）？

7. **体验账号防滥用**：手机号注册是否限制频次/设备数？
8. **管理后台要不要带"数据分析"**（销量/用户行为）—— 本期不做
9. **企业开票资质**：法人、税务号、银行账户等
10. **国际化**：是否支持英文版商城（参考 vention.io 的多语言）

## 附录 B：非本期但相关需求

1. **3D 仿真器边框全部去除**（OS 端）
2. **运行模式切换的菜单逻辑重构**（OS 端）
3. **夹爪属性面板交互调整**（OS 端）
4. **3D 场景里碰撞体可视化编辑**（绿框）
   - 鼠标右键 / 夹爪属性面板 → 触发「感知范围」配置
   - **不进入新界面**，在 3D 场景内直接编辑盒子大小
   - **不需为每个机械臂点位配置"夹什么物料"**——由物理引擎碰撞自动检测
5. **抓取逻辑简化**：抓取检测=「夹爪闭合 + 绿框与物料有碰撞」，**不需用户配置物料名称**
6. **精确抓取 vs 模糊抓取 选项**（工程师配置侧）
   - 模糊：只要检测到碰撞就夹住（适合粗动画演示）
   - 精确：必须在夹爪行程范围内正确形成才夹住（适合仿真）
   - 默认值需与设计师对齐
7. **抓取测试按钮**（高亮被检测到的物料）—— 用户调好绿框后，点测试即可看到能否夹住
8. **退出编辑模式用 ESC 快捷键**（绿框调节时）
