## 二、关键发现：新旧两套网站并存

### 2.1 新官网（营销风格，仅 2 个页面）

- 路径：`/`、`/about-contact.html`
- 技术栈：layui v2.11.2 + 自研静态资源（`/static/home/css/style.min.css`、`/static/home/js/extend.min.js`）
- 资源路径：`/uploads/upload/images/20260225/...`（2026 年 2 月上传的图）
- 字体：庞门正道标题体免费版

- 路径：`/terminal-gripper/`、`/4-axis-robots/`、`/6-axis-collaborative/`、`/medical-theme/`、`/3c-industrial-page/`、`/tearobot/`、`/about-us/`、`/application/` 等

- 风格虽然相似（红黑 + 庞门正道），但**导航菜单完全不一样**

## 三、导航菜单加载机制

### 3.0 2026-06-06 复核结论

最新线上页面仍然是同一套结构：

- 首页：`https://www.hitbot.cc/`
- 联系页：`https://www.hitbot.cc/about-contact.html`
- 主导航仍为 3 项：`产品中心` / `资源中心` / `关于慧灵`
- PC 与移动端导航仍各写一份：`.c-nav` 与 `.c-nav2-box`
- 主导航链接仍大量使用 `target="_blank"` 和 `rel="nofollow"`
- 搜索区仍是图标入口，HTML 中只看到 form / icon 结构，未形成完整站内搜索体验
- 页脚仍为 `#c-footer`，4 列仍为 `机器人` / `HITBOTOS系统` / `行业智能体` / `关于慧灵`

这意味着商城要做到“从官网切过来毫无察觉”，不能只改商城自身。官网侧也必须同步改 header / footer / mobile nav，并去掉商城相关链接的 `_blank` 行为。

### 3.1 HTML 结构（来自首页实际 DOM）

```html
<header id="c-header">
  <div class="container">
    <a class="c-logo" href="/">...</a>
    <div class="c-right-box">
      <ul class="c-nav">
        <!-- 主导航（PC 端） -->
        <li><a href=".../terminal-gripper/">产品中心</a></li>
        <li><a href=".../medical-theme/">资源中心</a></li>
        <li>
          <a href=".../about-contact.html">关于慧灵</a>
          <ul>
            <li><a href=".../about-us/">公司介绍</a></li>
            <li><a href=".../application/">新闻资讯</a></li>
            <li><a href=".../about-contact.html">联系我们</a></li>
          </ul>
        </li>
      </ul>
      <div class="c-nav2-box">...</div>
      <!-- 响应式移动导航 -->
      <div class="c-gn">搜索图标</div>
      <div class="c-switch"><i></i><i></i><i></i></div>
      <!-- 移动端汉堡按钮 -->
    </div>
  </div>
</header>
```

### 3.2 关键结论

| 维度           | 结论                                                                               |
| -------------- | ---------------------------------------------------------------------------------- |
| **菜单数据源** | **HTML 硬编码**，没有后端 API（如 `GET /api/nav`），没有数据库表，没有菜单管理后台 |
| **逐页复制**   | 每个页面**自己写一份** header（包括 mobile 版本 `.c-nav2-box`）—— 修改菜单需逐页改 |
| **子菜单**     | "关于慧灵"有 2 级下拉，纯 HTML/CSS 实现，无 JS 交互逻辑                            |
| **激活态**     | 当前页对应的菜单项有红色下划线（CSS 类控制，需手工加 class）                       |
| **搜索**       | 仅有搜索图标，**点击无反应**（没接搜索后端）                                       |
| **国际化**     | 旧站右上角有"中文/英文"切换，**指向 hitbotrobot.com**（英文国际站是另一套）        |

### 3.3 后端痕迹

- `POST /site/flow` 返回 **404** —— 本应有个流量/SEO 统计接口，但没配
- 整站用了百度统计（`hm.baidu.com`）和百度爱番番客服（`affim.cdn.bcebos.com`）—— 营销工具齐全，但**业务后端基本为空**

### 3.4 改造影响

要在导航里加"商城 / 在线体验 OS / 登录 / 购物车"4 个新入口：

- **必须逐个页面改 HTML**（包括新官网 2 个 + WordPress 站 8+ 个）
- **响应式 mobile 版的 `.c-nav2-box` 也要同步改**
- 没有"一次改、全局生效"的可能
- **建议**：要么把商城做在"新官网"上并迁所有页面到新模板（彻底告别 WordPress），要么把"新官网"反过来寄生进 WordPress（保留旧基础设施）—— **强烈建议前者**（详见第六节）

## 四、视觉设计规范

### 4.1 新官网设计语言

**首页（深色营销页）**：

- 背景：纯黑 / 深灰
- 主标题：超大白色无衬线字体（庞门正道）
- 副标题：白色小字
- 关键信息用大字号"留白"做视觉冲击
- 视频/产品图占满右侧

**关于慧灵（白底内容页）**：

- 背景：纯白
- 标题：黑色
- 表单字段：底部下划线，无边框（**与 PRD §7.2 "边框基本不用"完全一致**）
- 必填字段：红色 `*`
- 主按钮：深红（接近 `#B71C1C`/`#C00000`）长条矩形 + 白色文字 + 右上箭头
- 二级按钮 / 链接：黑色 + 红色 hover

**新官网 footer**（4 列产品分类）：

- 机器人 / HITBOTOS系统 / 行业智能体 / 关于慧灵
- 标题黑色 + 链接深灰
- "HITBOTOS系统"下 4 个子项**全部是死链 `javascript:void(0)`**（具身大脑/仿真训练/技能库/具身执行矩阵）——这 4 个二级页面都没做
- 友情链接列：淘宝店铺、汇加工、爱采购平台、机器人在线、1688店铺

### 4.2 旧官网设计语言（WordPress 站）

- 同样的红黑 + 庞门正道 + 极简
- 红色下划线 hover（首页产品分类标题）
- 红色小药丸按钮（如"eHand-6"）
- 客服：黄色"在线咨询" + 红色"请您留言"侧边栏（新站同样有）
- 整体调性 = **和 PRD §7 视觉规范的"颜色表 + 去边框"约束方向一致**

### 4.3 颜色提取（从截图比对）

| 用途     | 颜色                                           | 备注                   |
| -------- | ---------------------------------------------- | ---------------------- |
| 主品牌色 | 深红 `#B71C1C` ~ `#C00000`                     | 按钮、激活态、品牌点缀 |
| 主文字   | 纯黑 `#000000` / 深灰 `#333`                   | 标题、正文             |
| 次要文字 | 浅灰 `#999` / `#666`                           | 提示、placeholder      |
| 背景     | 白 `#FFFFFF`（默认） / 纯黑 `#000`（首页营销） | 极端两极               |
| 客服色   | 黄色（在线咨询按钮）/ 红色（留言侧边栏头部）   | 第三方爱番番固定色     |

### 4.4 字体

- **标题**：庞门正道标题体免费版（`PangMenZhengDaoBiaoTiTiMianFeiBan-2.ttf`）
- **正文**：浏览器默认无衬线（PingFang/Microsoft YaHei/stack）
- **图标**：内联 SVG + iconfont（`at.alicdn.com` 阿里图标库）

### 4.5 交互组件

- **按钮**：实心红底白字 / 黑色描边 / 文字按钮（3 种）
- **表单**：底部下划线 input（**零边框**风格）
- **卡片**：白底 + 极轻阴影 + 圆角约 4-8px
- **导航 hover**：底部红色下划线滑入
- **客服**：右下角浮动"在线咨询" + 左侧浮动"请您留言"面板
- **搜索图标**：右上角，纯图标

## 六、商城落地策略建议

### 6.1 嵌入位置决策（最重要的策略选择）

**方案 A：嵌入到新官网，并把现有 8+ 个 WordPress 旧页全部迁过来**（推荐）

- 改造成本：中等（一次性的页面迁移 + 模板改造）
- 长期收益：统一技术栈，菜单只改一处，**符合 PRD §1.2 "嵌入到新官网"**
- 风险：现有产品/SEO 链接可能短期受影响（要做 301 重定向）

**方案 B：商城独立子域**（如 `store.hitbot.cc`）

- 改造成本：低（两边完全解耦）
- 长期收益：互不影响，可独立技术栈
- 风险：和官网视觉割裂，登录态/购物车跨域问题，**违反 PRD §1.2 "嵌入到新官网"**

**方案 C：寄生在旧 WordPress 站**

- 改造成本：低（沿用现有基础设施）
- 长期风险：技术债严重（WordPress 维护、菜单硬编码、零后端），**强烈不推荐**

**推荐方案 A**。但需要分阶段执行：

- 第一阶段（M1-M2）：**只新增商城 + 关于慧灵 + 产品中心入口**到新官网，WordPress 旧页暂时保留（用 js 重定向或保留访问）
- 第二阶段（M3+）：把 WordPress 旧页逐步迁到新官网

### 6.2 视觉风格建议

**商城子页面**直接复用新官网 about-contact.html 的设计语言：

- 顶部导航完全沿用（白底 + 黑色字体 + 红色 hover/激活下划线）
- 字号、间距、圆角、按钮形态 100% 一致
- **强烈建议**：设计师出"颜色规范表格"时，**直接基于新官网现有色值**，不要重新定
- 物料三分类（标准件/加工件/参考件）用色建议：和品牌深红 `#B71C1C` + 中性灰 `#666` + 警示橙 `#FF6B35` 做对比（**和 PRD §3.3 完全契合**）
- 字体：复用庞门正道（标题）+ 默认无衬线（正文）

**首页商品列表/产品详情页**可直接借鉴 `/terminal-gripper/`（WordPress 旧站）的**产品卡片网格布局**——红黑配色、小药丸标签、规范参数列表，是已经验证过的样式。

### 6.3 菜单新增方案

2026-06-06 复核后，建议不要粗暴新增 4 个顶级 `<li>`，否则会破坏官网当前极简 3 项导航的节奏。更稳的接入方式如下：

1. **产品中心顶级入口指向商城商品列表**
   - 文案仍叫“产品中心”，避免官网突然多出“商城”导致品牌感变电商。
   - 推荐路径：`/store/products`，由官网网关 / rewrite 映射到本项目。
   - 旧站的 `/terminal-gripper/`、`/4-axis-robots/`、`/6-axis-collaborative/` 改为产品中心下拉项或 footer 深链。

2. **产品中心下拉第一项放“官方商城 / 在线选购”**
   - 如果市场侧仍希望保留旧产品资料页作为顶级产品中心，则给“产品中心”加下拉：
     - 官方商城
     - 机器手
     - 机器臂
     - 人形机器人
   - “官方商城”必须排第一，且同标签页打开。

3. **右侧工具区放购物车与登录**
   - 位置：搜索图标之后，移动端汉堡之前。
   - 形态：图标按钮为主，登录可做文字 / 图标状态切换。
   - 购物车出现角标后才有“商城存在感”，但不要在主导航再重复一项“购物车”。

4. **在线体验 OS 放在 HITBOTOS 系统语境中**
   - 首页：HITBOTOS / OS 介绍区加按钮或文字链。
   - Footer：`HITBOTOS系统` 列第一项替换为 `在线体验 OS`，再保留具身大脑 / 仿真训练等概念项。
   - 不建议把“在线体验 OS”放成主导航顶级项，当前 PRD 也说优先级中等偏低。

移动端 `.c-nav2-box` 同步加。

**工作清单**：

- 找所有页面（共 13 个），逐个改 header HTML
- 同步改 `.c-nav` 和 `.c-nav2-box`，不要只改 PC 端
- 商城相关入口去掉 `target="_blank"`，站外/旧 WordPress 资料页才保留 `_blank`
- 产品路径做一张 URL 映射表，至少覆盖官网入口、商品列表、商品详情、购物车、登录、在线体验 OS
- 提交后等 Growthman 帮忙部署，或者**接管建站公司，自己部署**
- 第一版至少交付：商城商品列表、商品详情、购物车、登录/个人中心占位、在线体验 OS 占位

### 6.4 不建议做的事

- ❌ **不要沿用现有"灰度黄"客服色**作为商城主色（和爱番番冲突）
- ❌ **不要用 WordPress 插件做商城**（性能/扩展性/账号体系都受限）
- ❌ **不要给现有 8+ 个 WordPress 页面加新导航**（一改全改，避免遗漏）
- ❌ **不要重新设计颜色规范**（直接基于现有红黑白出表）

## 七、给后续项目的提醒

> "涉及到的很多个现成的项目，一个个来看"—— 这次是官网，**结论要先记住**：
>
> 1. **菜单是硬编码 HTML** —— 任何项目只要动到全局导航，就知道要在哪改
> 2. **新官网只有 2 个页面** —— 任何"全站"功能（如购物车、登录、个人中心）的接入面很小
> 3. **建站方是 Growthman** —— 如果不想被外包卡脖子，下一步建议**接管代码 + 部署权限**
> 4. **后台基本为空**（POST /site/flow 404）—— 后续需要从零搭后端，**不用考虑和现有后端兼容**
