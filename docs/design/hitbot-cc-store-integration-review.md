# hitbot.cc 官网接入商城整合审查

> 日期：2026-06-06
> 对象：`https://www.hitbot.cc/`、`https://www.hitbot.cc/about-contact.html`、本项目 `http://localhost:3002/zh`
> 目标：判断商城如何接入 hitbot.cc，做到从官网切入后视觉、导航和行为尽量无感。

> **2026-06-06 更新**：本文件记录的是上一版整合审查结论，其中“官方商城”“在线体验 OS”进入官网导航/footer 的建议已废止。当前执行口径是：商城页头/页尾只参考新版 hitbot.cc 官网；商城页头右侧只保留购物车和用户/个人中心入口；不做单独商城首页，根入口直达商品采购页。

---

## 一、结论先行

商城不应该作为一个突兀的“新站”挂出去。最稳的方案是：

1. 官网主导航仍保留 `产品中心` 这个顶级词。
2. `产品中心` 默认落到商城商品列表，推荐公开路径为 `/store/products`。
3. 旧官网产品资料页保留为 `产品中心` 下拉或 footer 深链。
4. 购物车、登录放在右侧工具区，不放进主导航。
5. `在线体验 OS` 放进 HITBOTOS 系统语境，不抢主导航顶级位置。
6. 商城页头页尾必须复制官网行为，而不是只复制 logo 和颜色。

当前本项目已经接近官网的基础视觉，但还没有达到“毫无察觉”。主要露馅点是移动端无主导航入口、官网链接打开方式不一致、footer 微信/移动折叠行为不一致、官网客服组件缺失，以及商城首页首屏仍偏“独立电商页”。

## 二、入口链接放在哪里

### 2.1 PC 顶部导航

推荐结构：

```text
HITBOT logo
产品中心
  ├─ 官方商城 / 在线选购  -> /store/products
  ├─ 机器手              -> /terminal-gripper/
  ├─ 机器臂              -> /4-axis-robots/
  └─ 人形机器人          -> /6-axis-collaborative/
资源中心                 -> /medical-theme/
关于慧灵
  ├─ 公司介绍            -> /about-us/
  ├─ 新闻资讯            -> /application/
  └─ 联系我们            -> /about-contact.html
搜索 购物车 登录
```

如果只能做最小改动：

- 把官网 `产品中心` 从 `https://www.hitbot.cc/terminal-gripper/` 改为 `/store/products`。
- 在 footer 的 `机器人` 列新增第一项 `官方商城`。
- 搜索图标右侧新增购物车图标。

不推荐新增顶级 `商城`。原因是官网当前是 3 项极简导航，突然多一个“商城”会让品牌站变成交易站；用户也更自然地把明码标价商品理解为“产品中心”的升级，而不是新业务。

### 2.2 首页内容区入口

首页首屏的“了解更多”不建议直接改成商城主 CTA，因为官网首页的使命仍是品牌叙事。更合适的位置：

- 首页产品/机器人相关区块：增加 `在线选购` 或 `查看全部产品`。
- HITBOTOS 系统区块：增加 `在线体验 OS`。
- 底部 footer：`HITBOTOS系统` 列第一项改为 `在线体验 OS`。

### 2.3 移动端入口

移动端必须同步改 `.c-nav2-box`。官网现在 PC 与移动菜单是两份 HTML，漏改移动端会直接导致“官网 PC 有商城，手机找不到商城”。

移动端推荐顺序：

```text
产品中心
  官方商城
  机器手
  机器臂
  人形机器人
资源中心
关于慧灵
购物车
登录 / 个人中心
在线体验 OS
```

## 三、路径与打开方式

### 3.1 推荐公开路径

| 功能                | 推荐路径                     | 备注                               |
| ------------------- | ---------------------------- | ---------------------------------- |
| 商城首页 / 商品列表 | `/store/products`            | 官网 `产品中心` 默认落点           |
| 商品详情            | `/store/products/{slug}`     | slug 需要和真实型号 / SEO 策略统一 |
| 购物车              | `/store/cart`                | 顶部购物车入口                     |
| 登录                | `/store/login`               | 后续可接统一账号                   |
| 个人中心            | `/store/account`             | 登录后替换登录入口                 |
| 在线体验 OS         | `/try-os` 或 `/store/try-os` | 取决于是否归入商城应用             |

当前项目路由是 `/zh/products`、`/en/products`。如果要无感嵌入官网，建议在部署层做 rewrite：

```text
/store/products        -> /zh/products
/store/products/:slug  -> /zh/products/:slug
/store/cart            -> /zh/cart
```

不要把官网入口直接写成 `/zh/products`。官网本身没有 `/zh` 语言前缀，用户会明显感知到技术栈切换。

### 3.2 打开方式

商城相关入口必须同标签页打开：

- 不加 `target="_blank"`
- 不加 `rel="nofollow"`
- active 状态与官网一致，`产品中心` 高亮

旧 WordPress 资料页可以保留 `_blank`，但从商城回官网的“关于慧灵 / 资源中心”建议也同标签页打开，整体体验更像一个站。

## 四、页头对齐审查

### 4.1 已对齐

- logo 素材已放在 `public/hitbot/logo.svg`。
- 桌面 header 高度已做到 `100px`，移动端 `60px`，接近官网。
- container 已使用 `w-[96%] max-w-[1840px]`，与官网 header 接近。
- 桌面导航字号使用 `text-base`，实际 token 为 16px。
- 搜索图标容器宽度 `79px` 与官网接近。

### 4.2 仍需调整

| 优先级 | 问题                                       | 影响                                               | 建议                                                                        |
| ------ | ------------------------------------------ | -------------------------------------------------- | --------------------------------------------------------------------------- |
| P1     | 移动端没有汉堡菜单                         | 手机用户从官网进商城后找不到产品中心/资源/关于入口 | 复刻官网 `.c-switch` / `.c-nav2-box` 行为，至少做 Sheet 菜单                |
| P1     | 官网是 `fixed` header，商城是 `sticky`     | 滚动行为不一致，首屏切换能感到差异                 | 商城公共页改成 fixed，主内容补对应 top padding；或官网侧统一成 sticky       |
| P1     | 官网商城入口若继续 `_blank` 会开新标签     | “无感切换”直接失败                                 | 商城、购物车、登录、体验 OS 均同标签页打开                                  |
| P2     | 右侧工具区比官网多语言、主题切换、登录文字 | 视觉更像应用 Demo，不像官网原生页                  | 公网商城隐藏主题切换；语言切换按官网国际站策略处理；登录可压缩为图标/登录态 |
| P2     | logo 在移动端偏小且右侧图标拥挤            | 首屏品牌感弱，操作区密度高                         | 移动端保留 logo + 搜索/购物车 + 汉堡，其余放菜单内                          |

## 六、页面体验审查

### 6.1 首页

当前本地首页比官网更像一个商城 Demo。主要问题：

- 首屏右侧图片被放进圆角卡片，官网首页更偏沉浸式大图/视频，联系页则是干净白底内容，不太使用“卡片 hero”。
- 首屏商品 CTA 文案是 `浏览商品中心`，但导航也叫 `产品中心`，文案不够统一。
- `在线体验 OS` 按钮存在，但没有落地链接，容易造成空承诺。
- 首页商品卡片直接大红按钮，很有电商感；官网产品页更克制，偏产品资料和咨询。

建议：

- 如果商城入口落在 `/store/products`，首页可以退居为商城概览，少做营销 hero。
- 如果保留商城首页，首屏改为“产品中心 / 在线选购”的官网式内容页，不用电商模板感的大卡片。
- `在线体验 OS` 未交付前做占位页，不要按钮无行为。

### 6.2 商品列表

当前商品列表清晰，但还有三点会影响官网融合：

- 筛选区是白色卡片，产品卡也是白色卡片，卡片感偏强。
- 价格和红色加入购物车按钮视觉权重很高，适合交易，但不够 B2B 克制。
- 移动端商品图比例和商品卡高度差异明显，部分参考件图片区域显得空。

建议：

- 列表页作为“产品中心”落点可以保留价格，但把按钮弱化为详情页主转化，列表只保留 `查看详情` / hover 加购。
- 参考件和定制件需要更清楚的“不直接售卖”解释，避免用户误以为缺货。
- 移动端图片容器设稳定最小高度，并为缺图 / 横图做专门裁切策略。

### 6.3 商品详情

详情页基本可用，适合 B2B 信息呈现。上线前重点补：

- 商品 slug 与官网/SEO 映射表。
- 真实型号、旧产品页、商城详情页之间的 301 / canonical 策略。
- 参数表字段完整度。
- “联系询价 / 加入购物车 / 对公采购”之间的文案层级。

## 七、客服与第三方组件

官网有两个明显的咨询入口：

- 右侧蓝色 `在线咨询`
- 左下 / 左侧红色 `请您留言`

商城目前没有这些组件。从官网进入商城后，咨询入口消失，会让用户觉得站点切换了。

建议：

- 公共浏览页、商品列表、商品详情保留与官网一致的咨询组件。
- 购物车、结算页可以弱化或收起，避免遮挡关键流程。
- 如果暂时不接爱番番脚本，至少做同位置的“咨询销售”占位入口。

## 九、优先级改造清单

### P1 必做

1. 官网侧把 `产品中心` 接到商城商品列表，或下拉第一项增加 `官方商城`。
2. 官网 `.c-nav2-box` 同步增加商城、购物车、登录、在线体验 OS。
3. 商城移动端补主导航菜单。
4. 商城相关入口同标签页打开，去掉 `_blank`。
5. 桌面 footer 微信二维码改为官网式小图标 hover，移动端 footer 改折叠。
6. 上线前定义 `/store/*` 到当前 `/zh/*` 的 rewrite / redirect 策略。

### P2 应做

1. 公网商城隐藏主题切换，语言入口按官网国际站策略处理。
2. 购物车、登录图标样式与官网搜索图标统一。
3. 商品列表弱化红色满宽按钮，减少模板电商感。
4. 接入官网咨询组件或提供同位置咨询入口。
5. `在线体验 OS` 增加真实占位页或禁用态说明。

### P3 可后续

1. 统一旧 WordPress 产品页与商城商品详情页的 SEO / canonical。
2. 梳理真实商品图比例，补缺图策略。
3. 把 footer / header token 做成更明确的官网专用组件规范。
